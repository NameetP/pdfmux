# pdfmux-bench corpus

The corpus is the benchmark. A parser that aces clean digital-native PDFs and
falls apart on a skewed scan of a two-column paper with a merged-cell table is
not a good parser — it just had an easy test. This corpus is built to be hard on
purpose, and to be **legally clean**: only public-domain or openly-licensed
documents, referenced by a manifest so we never redistribute anything we
shouldn't.

## Categories

Each category targets a distinct failure mode. An engine's per-category scores
are on the leaderboard, so weaknesses can't hide in an average.

| Category | Directory | What breaks weak engines |
|---|---|---|
| Digital-native | `digital-native/` | baseline — text layer present; anyone should do well |
| Scanned | `scanned/` | no text layer; pure OCR quality |
| Complex tables | `complex-tables/` | merged cells, spanning headers, nested tables |
| Multi-column academic | `academic/` | column order, cross-column reading order, references |
| Forms / checkboxes | `forms/` | field labels, checkbox state, key-value pairs |
| Handwriting | `handwriting/` | cursive / historical script OCR |
| Arabic / RTL | `rtl/` | right-to-left order, bidi, ligatures |
| Degraded scans | `degraded/` | skew, bleed-through, low DPI, historical newsprint |

## Layout

```
corpus/
├── README.md
├── manifest.json          ← the source of truth: what docs, from where, license
├── fetch_corpus.py        ← downloads each doc's PDF from its manifest URL
├── <category>/
│   ├── <doc-id>.pdf        ← downloaded by fetch_corpus.py (git-ignored)
│   └── <doc-id>.gt.md      ← ground-truth Markdown (committed; this is the label)
```

- **PDFs are NOT committed** (`.gitignore` excludes `corpus/**/*.pdf`). They are
  fetched from their canonical public URLs at run time. This keeps the repo small
  and sidesteps redistribution entirely — we ship pointers and labels, not the
  source documents.
- **Ground-truth `.gt.md` files ARE committed.** They are the benchmark labels.
  Building them is the real work of the corpus (see "Ground truth" below).

## The manifest

`manifest.json` is a list of document records. Schema:

```jsonc
{
  "schema_version": "1.0",
  "documents": [
    {
      "id": "irs-f1040-2024",          // unique, filesystem-safe
      "category": "forms",             // one of the categories above
      "title": "IRS Form 1040 (2024)",
      "url": "https://www.irs.gov/pub/irs-pdf/f1040.pdf",  // canonical source; null if not yet pinned
      "source": "US Internal Revenue Service",
      "license": "public-domain",      // SPDX-ish tag or "public-domain"
      "license_url": "https://www.usa.gov/government-works",
      "redistributable": true,          // may the SOURCE pdf be redistributed?
      "needs_pin": false,               // true = url/sha not yet finalized; fetcher skips
      "sha256": null,                   // filled in when the doc is pinned (integrity check)
      "pages": "1-2",                   // optional page range to benchmark (keeps runs fast)
      "gt_status": "pending",           // "pending" | "done" — is <id>.gt.md built + reviewed?
      "notes": "Two-page tax form; checkboxes + key-value fields."
    }
  ]
}
```

`fetch_corpus.py` downloads every record whose `url` is set and `needs_pin` is
`false`, into `<category>/<id>.pdf`, verifying `sha256` when present.

## Starter manifest

The committed `manifest.json` ships with **real, verifiable public-domain and
openly-licensed sources** across categories:

- **IRS forms** (Form 1040, W-9, W-4) — US government works, public domain,
  stable URLs at `irs.gov/pub/irs-pdf/`. Forms + checkboxes + key-value.
- **arXiv papers** (Attention Is All You Need, BERT, InstructGPT) — canonical
  `arxiv.org/pdf/<id>` URLs; multi-column academic layout with tables/equations.
- **SEC EDGAR filings** — public company filings via EDGAR full-text search;
  dense financial tables. (Specific accession `needs_pin` by a contributor.)
- **Library of Congress / Chronicling America** — public-domain historic
  newspapers and manuscripts for degraded scans and handwriting. (`needs_pin`.)
- **Arabic Wikipedia PDF export** — CC-BY-SA, deterministic REST PDF endpoint,
  for RTL/bidi.

Records marked `needs_pin: true` name a real, authoritative source collection but
leave the exact document URL + sha256 for a contributor to finalize — we do not
invent direct links we haven't verified. The fetcher skips them until pinned.

## Ground truth

Ground truth is the hard, valuable part — and where contributors add the most
value. For each document:

1. Produce a faithful Markdown rendering of the document's content: headings as
   `#`, tables as GitHub-flavored Markdown tables, reading order top-to-bottom
   and (for multi-column) column-by-column, checkboxes as `[x]`/`[ ]`.
2. **Human-review it.** Ground truth generated by any single engine is biased
   toward that engine. Start from an engine if you like, but a human must
   correct it. Note the process in the record's `notes`.
3. Save as `<category>/<id>.gt.md` and set `gt_status: "done"`.

Ground-truth guidelines (encoding, table conventions, RTL handling) live in
[CONTRIBUTING.md](../CONTRIBUTING.md).

## Adding a document

1. Find a **public-domain or openly-licensed** document that stresses a category.
   No copyrighted material, no "found a PDF online" — the license must be clear
   and redistributable-as-a-pointer.
2. Add a record to `manifest.json` (fill `url`, `license`, `sha256` if you can).
3. Build and human-review `<category>/<id>.gt.md`.
4. Open a PR. CI validates the manifest and re-runs the OSS engines.

## Licensing stance

Only documents whose license we can name and verify enter the corpus. US federal
government works (IRS, SEC, LoC, GAO, NASA) are public domain under 17 U.S.C.
§105. arXiv and Wikipedia content is redistributable under their stated terms
(recorded per-record). If a document's license is unclear, it does not go in.

## GT-0 — the frozen, pre-registered corpus

`manifest.json` (schema 1.1) is the **GT-0** build: **24 documents, 3 per category**, every
`sha256` pinned and every `gt_status: "done"`. Each record carries
`gt_provenance: "rendered-image transcription, non-contestant"` — the ground truth was authored
by reading a 200-DPI PyMuPDF render of the benchmarked page(s) and transcribing by hand, **never
from any extractor's output**.

- **Adversarial coverage** (tagged in each record's `stressors`): 5 pure image-only scans
  (telegram, two IMTFE typescripts, two cursive manuscript letters); a CID/Type3-font document
  (`arxiv-2203.02155-instructgpt`); and a rotated/landscape multi-column document
  (`irish-census-1926`, a 1663×728 census schedule with 15+ columns).
- **[`PREREGISTRATION.md`](PREREGISTRATION.md)** freezes the manifest SHA-256 and a ground-truth
  content digest *before* any pdfmux score was computed — the commit adding it precedes the
  validation-results commit.
- **[`agreement-report.md`](agreement-report.md)** gives inter-annotator agreement for the 8
  double-annotated documents (one per category), computed by [`agreement.py`](agreement.py)
  over `annotations/pass1` vs `annotations/pass2`.
- The measured false-positive / false-negative rates of `verify_extraction` on this corpus live
  in [`../VERIFIER-VALIDATION.md`](../VERIFIER-VALIDATION.md), regenerated by
  [`../validate_verifier.py`](../validate_verifier.py).

`pages` selects the benchmarked page range (1-indexed); multi-page ground truth uses
`<!-- page: N -->` markers.
