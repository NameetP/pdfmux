# Contributing to pdfmux-bench

There are three ways to contribute, in rough order of value to the benchmark:

1. **Add an engine** — write an adapter so your tool (or a tool you use) gets scored.
2. **Add a document** — grow the corpus with a hard, license-clean document + ground truth.
3. **Improve the methodology** — sharpen a metric, or replace the TEDS approximation.

The one rule that governs all of them: **radical honesty**. No fabricated numbers, no home-field advantage, publish losses. If a change would make pdfmux look better without making the measurement more accurate, it doesn't land.

---

## 1. Add an engine

> **If you think your tool wins, send a PR with an adapter. We will run it and publish the result — win or lose.**

An adapter is one file. Copy the shape of an existing one:

- Closed API → model it on [`adapters/reducto.py`](adapters/reducto.py) / [`adapters/mistral_ocr.py`](adapters/mistral_ocr.py).
- Local OSS → model it on [`adapters/docling.py`](adapters/docling.py) / [`adapters/mineru.py`](adapters/mineru.py).

```python
# adapters/myengine.py
from pathlib import Path
from .base import Adapter

class MyEngineAdapter(Adapter):
    name = "myengine"                 # leaderboard id (filesystem-safe)
    label = "MyEngine"                # display name
    homepage = "https://myengine.dev"
    license = "Apache-2.0"            # or "closed-api", "MIT", ...
    env_var = "MYENGINE_API_KEY"     # None if it runs locally with no key

    def available(self) -> bool:
        # Default checks env_var. Override for local packages:
        # return importlib.util.find_spec("myengine") is not None
        return super().available()

    def extract(self, pdf_path: Path) -> str:
        # Do the real work. Return Markdown. RAISE on failure.
        ...
```

Then register it in [`adapters/__init__.py`](adapters/__init__.py) (import + append to `ALL_ADAPTERS`; add to `OSS_LOCAL` if it's local and key-free).

**Adapter rules (enforced by review):**

- Return the engine's **real** output. Never fall back to another engine, never synthesize Markdown, never swallow an error into empty output.
- Missing key / package → `available()` returns `False` → the harness **skips** you with a clear message. Never fake a row.
- A real failure → **raise**. The harness records it as an error row. That's data, not a crash.
- Document the real API shape (endpoint, auth env var, request/response mapping) in the module docstring, even if the call is gated behind a key.
- Don't add a heavyweight dependency to `requirements.txt` core — make it an optional/commented install line.

## 2. Add a document

The corpus is where the benchmark's credibility lives. See [`corpus/README.md`](corpus/README.md) for the full spec. In short:

1. **License first.** Only public-domain or openly-licensed documents whose license you can name and link. US federal government works (IRS, SEC, LoC, GAO, NASA), arXiv, Wikipedia, Project Gutenberg are safe; "a PDF I found" is not.
2. Add a record to [`corpus/manifest.json`](corpus/manifest.json) with `id`, `category`, `url` (canonical source), `license`, `license_url`, and — if you can compute it — `sha256`. If you can't finalize the exact URL, set `needs_pin: true` and name the real source collection; don't invent a direct link.
3. **Build ground truth** and human-review it (below). Save as `corpus/<category>/<id>.gt.md`, set `gt_status: "done"`.
4. Open a PR.

### Ground-truth conventions

Ground truth is Markdown. Consistency matters more than perfection — apply these uniformly:

- **Headings**: `#`/`##`/`###` matching the document's visual hierarchy.
- **Reading order**: top-to-bottom; for multi-column, fully read the left column before the right (Western) — the order a human reads in. This is what `reading_order` scores against.
- **Tables**: GitHub-flavored Markdown tables. One header row, one `---` separator, then data rows. Represent merged/spanning cells by repeating the value across the spanned cells. Keep cell text trimmed.
- **Checkboxes**: `[x]` checked, `[ ]` unchecked, followed by the label text.
- **Key-value fields** (forms): `**Label:** value` on one line.
- **RTL (Arabic/Hebrew)**: transcribe in logical (reading) order. Keep the source script; don't transliterate. Note any bidi digit handling in the record `notes`.
- **Don't invent content.** If text is illegible in a scan, mark it `[illegible]` rather than guessing. Ground truth must be faithful, not fluent.

> **Human review is mandatory.** You may bootstrap ground truth from an engine, but a human must correct it before it's committed. Ground truth generated by one engine and left uncorrected biases the whole board toward that engine. State your process in the record's `notes`.

## 3. Improve the methodology

- **Metrics** live in [`scoring.py`](scoring.py) with zero dependencies and zero pdfmux imports — keep it that way so scoring runs anywhere and treats every engine identically.
- **Tracked good-first-issue — full TEDS.** `table_teds` is currently a grid-structure similarity, not a byte-identical PubTabNet TEDS (APTED tree-edit distance over parsed table HTML). A faithful APTED TEDS implementation (behind the same `table_teds(extracted, ground_truth) -> float` signature) is a high-value contribution. Document any correlation study against the reference TEDS in the PR.
- **Weights** (`DEFAULT_WEIGHTS`) are fixed and printed on every leaderboard. Changing them is a methodology change: bump `VERSION`, keep historical `results/` files, and explain the rationale in the PR. Never tune weights to change who wins.

## Running locally before you PR

```bash
pip install -r requirements.txt
python corpus/fetch_corpus.py --list        # see corpus status
python corpus/fetch_corpus.py               # fetch pinned docs
python run_bench.py --oss-only              # run what CI runs (no keys needed)
python leaderboard.py --results results/latest.json
```

CI (`.github/workflows/bench.yml`) runs the OSS engines on the public corpus and regenerates the leaderboard on every PR. Closed-API engines are validated by maintainers with keys.

## Code of conduct & neutrality

Be honest and be kind. The benchmark's whole value is that it can't be gamed because anyone can rerun it. Protect that. If you spot a way pdfmux (or any engine) is getting an unfair edge, **open an issue** — that's the most valuable contribution there is.
